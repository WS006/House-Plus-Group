## 🎯 关键 SEO 指标

### 1. 索引覆盖率

**目标：** 100% 的公开页面被索引

```
✅ 成功指标：
- 10 个页面都显示 "有效"
- 无 "已排除" 页面
- 无 404 或重定向错误

⚠️ 警告指标：
- 页面显示 "已排除"
- 出现 404 错误
- 出现重定向链
```

**检查步骤：**
1. 打开 GSC → 覆盖率
2. 查看 "有效" 页面数量
3. 如果 < 10，点击 "错误" 查看详情
4. 根据错误类型采取行动

### 2. 搜索性能

**目标：** 持续增加点击次数和展示次数

```
关键指标：
- 点击次数：目标 > 100/月
- 展示次数：目标 > 1000/月
- 平均排名：目标 < 20
- 点击率 (CTR)：目标 > 3%
```

**改进策略：**
- 优化 Meta 描述，提高 CTR
- 为高排名页面添加 FAQ Schema
- 创建更多原创内容
- 改进页面加载速度

### 3. 核心网页生命周期 (Core Web Vitals)

**目标：** 所有指标都达到 "良好" 水平

| 指标 | 良好 | 需要改进 | 不良 |
|------|------|--------|------|
| LCP (加载) | < 2.5s | 2.5-4s | > 4s |
| FID (交互) | < 100ms | 100-300ms | > 300ms |
| CLS (视觉稳定) | < 0.1 | 0.1-0.25 | > 0.25 |

**改进方法：**
1. 使用 PageSpeed Insights 检查
2. 优化图片大小和格式
3. 减少 JavaScript 体积
4. 启用浏览器缓存
5. 使用 CDN 加速

## 🔧 常见 SEO 问题排查

### 问题 1：页面未被索引

**症状：** GSC 显示 "已排除" 或 "发现但未索引"

**排查步骤：**

```
1. 检查 robots.txt
   - 访问 https://www.houseplus.com.ng/robots.txt
   - 确保页面未被 Disallow

2. 检查 noindex 标签
   - 打开页面源代码
   - 搜索 "noindex"
   - 如果存在，删除它

3. 检查页面质量
   - 页面是否有足够内容？
   - 是否是重复内容？
   - 是否有 canonical 标签？

4. 请求重新索引
   - GSC → URL 检查
   - 输入页面 URL
   - 点击 "请求编入索引"
```

### 问题 2：排名下降

**症状：** 页面排名突然下降

**排查步骤：**

```
1. 检查是否有算法更新
   - 查看 Google 搜索中心博客
   - 对比排名下降时间

2. 检查页面是否被修改
   - 最近是否改动了内容？
   - 是否删除了重要链接？
   - 是否改变了 Meta 标签？

3. 检查竞争对手
   - 竞争对手是否发布了更好的内容？
   - 是否有新的反向链接？

4. 改进策略
   - 更新页面内容
   - 添加新的反向链接
   - 改进页面速度
   - 增加用户参与度
```

### 问题 3：爬虫错误

**症状：** GSC 显示 4xx 或 5xx 错误

**排查步骤：**

```
1. 识别错误类型
   - 404：页面不存在
   - 403：禁止访问
   - 500：服务器错误

2. 对于 404 错误
   - 检查 GitHub Pages 路由配置
   - 验证 404.html 重定向逻辑
   - 测试页面是否可访问

3. 对于 403 错误
   - 检查 robots.txt 配置
   - 检查服务器权限
   - 检查 IP 白名单

4. 对于 500 错误
   - 检查服务器日志
   - 重启 GitHub Pages 构建
   - 检查依赖项是否正确
```
